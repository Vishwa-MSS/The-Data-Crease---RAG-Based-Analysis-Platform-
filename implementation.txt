# 🎯 IMPLEMENTATION GUIDE - Cricket Match Explorer

## 📋 Complete Step-by-Step Implementation

---

## PHASE 1: Initial Setup (5-10 minutes)

### Step 1.1: Set Up Python Environment

```bash
# Create project directory
mkdir cricket-explorer
cd cricket-explorer

# Create virtual environment
python -m venv venv

# Activate virtual environment
# On Windows:
venv\Scripts\activate
# On Mac/Linux:
source venv/bin/activate
```

### Step 1.2: Install Dependencies

```bash
# Install all required packages
pip install -r requirements.txt
```

**Expected packages to be installed:**
- streamlit==1.31.0
- pandas==2.1.4
- cricsummary==0.1.5
- matplotlib==3.8.2
- seaborn==0.13.0
- plotly==5.18.0
- sqlalchemy==2.0.25
- python-dateutil==2.8.2
- Pillow==10.2.0
- openpyxl==3.1.2

---

## PHASE 2: Data Preparation (10-30 minutes, depending on data size)

### Step 2.1: Download Cricket Data

1. Visit: https://cricsheet.org/downloads/
2. Download desired datasets (e.g., "T20 Internationals - all matches")
3. Extract the downloaded zip file
4. You'll get folders with JSON files

### Step 2.2: Organize Data Structure

Create the folder structure in your project:

```bash
# Create data directories
mkdir -p data/T20_Internationals
mkdir -p data/One_Day_Internationals
mkdir -p data/tests_json
mkdir -p data/Multiday_Matches
mkdir -p data/One_Days
mkdir -p data/Non_Official_T20

# Also create cache and exports directories
mkdir -p cache
mkdir -p exports
```

### Step 2.3: Copy Data Files

Move/copy your extracted JSON files into the appropriate folders:

```
data/
├── T20_Internationals/     ← Copy T20I JSON files here
├── One_Day_Internationals/ ← Copy ODI JSON files here
├── tests_json/             ← Copy Test match JSON files here
└── ... (other folders as needed)
```

**Example:**
```bash
# If you downloaded T20I data
cp /path/to/downloaded/t20s/*.json data/T20_Internationals/
```

---

## PHASE 3: First Launch (2-5 minutes)

### Step 3.1: Start the Application

```bash
streamlit run frontend/app.py
```

**What happens:**
- Streamlit server starts
- Browser opens automatically at http://localhost:8501
- You see the landing page

### Step 3.2: Index Your Data

1. On the landing page, you'll see a warning: "No data found in database"
2. Click the **"📥 Index Data Now"** button
3. Wait for the indexing process to complete

**Progress indicators:**
- You'll see: "Indexing data... This may take several minutes"
- Console shows progress: "Processing 100/1000..."
- On completion: "✅ Successfully indexed X matches!"

**Indexing times:**
- 100 matches: ~10 seconds
- 1,000 matches: ~1-2 minutes
- 10,000 matches: ~10-15 minutes
- 30,000+ matches: ~30-45 minutes

### Step 3.3: Verify Setup

After indexing completes:
- You should see: "✅ Database Ready! X matches indexed"
- Click **"🚀 Start Exploring Matches"**
- You're ready to use the application!

## PHASE 4: Using the Application

### 4.1: Basic Search

1. **Open the sidebar** (if collapsed, click the > arrow)
2. **Set filters:**
   - Format: T20 Internationals
   - Teams: India
   - Date Range: 2024-01-01 to 2024-12-31
3. **Click "🔍 Apply Filters"**
4. **View results** in table or card view

### 4.2: View Ball-by-Ball Data

1. **Select a match** from search results
2. **Click "🏏 View Ball-by-Ball Data"**
3. **Explore the data:**
   - View all deliveries
   - Filter by innings
   - See top batters and bowlers
4. **Download:** Click "📥 Download Full Dataset (CSV)"

### 4.3: Perform Match Analysis

1. **Select a match** from search results
2. **Click "📊 View Match Analysis"**
3. **Choose analysis type:**

   **Match Info Tab:**
   - Click "📄 Get Match Info" to see metadata
   - Download as JSON

   **Scorecard & Summary Tab:**
   - Click "📊 Get Scorecard - Team 1" for batting scorecard
   - Click "🎯 Fall of Wickets - Team 1" for wicket analysis
   - Click "📈 Extras - Team 1" for extras breakdown
   - Repeat for Team 2

   **Visualizations Tab:**
   - Click "Generate Worm Chart" to see cumulative runs
   - Select team and click "Generate Manhattan Chart"
   - Download charts as PNG

   **Advanced Stats Tab:**
   - View batting analysis (top scorers, strike rates)
   - View bowling analysis (wickets, economy rates)

### 4.4: Export and Download

Every analysis provides download options:
- **CSV files** - For tabular data
- **PNG images** - For charts
- **JSON files** - For raw data

Downloads are saved with descriptive names:
- `matchid_team1_vs_team2_scorecard.csv`
- `matchid_worm_chart.png`
- `matchid_match_info.json`

## PHASE 6: Maintenance and Updates

### 6.1: Adding New Data

When you get new match files:

```bash
# Copy new files to appropriate folder
cp new_matches/*.json data/T20_Internationals/

# Re-index in the application
# Option 1: In UI - Click "🔄 Re-index Data"
# Option 2: Command line
python -m backend.ingestion
```

### 6.2: Database Reset

If you need to start fresh:

```bash
# Delete the database
rm cricket_matches.db

# Restart the application and re-index
streamlit run frontend/app.py
```

### 6.3: Clear Cache

If experiencing issues:

```bash
# Clear cached CSV files
rm -rf cache/*

# Clear exports
rm -rf exports/*
```

## 🔍 Troubleshooting Common Issues

### Issue 1: "No matches found"

**Solution:**
- Check that JSON files are in the correct folders
- Verify folder names match exactly (case-sensitive)
- Re-index data
- Check JSON files are valid Cricsheet format

### Issue 2: Charts not displaying

**Solution:**
```bash
# Reinstall matplotlib
pip uninstall matplotlib
pip install matplotlib

# Restart Streamlit
```

### Issue 3: Slow performance

**Solution:**
- Use more specific filters to reduce results
- Let the cache build up (improves over time)
- Index smaller datasets initially

### Issue 4: Import errors

**Solution:**
```bash
# Ensure all dependencies are installed
pip install -r requirements.txt

# If still issues, try:
pip install --upgrade pip
pip install -r requirements.txt --force-reinstall
```

### Issue 5: cricsummary errors

**Solution:**
```bash
# Install cricsummary with system packages flag
pip install cricsummary --break-system-packages

# Or use specific version
pip install cricsummary==0.1.5
```

## 📊 Project File Structure Explained

### Backend Files

**database.py**
- Manages SQLite database
- Stores match metadata
- Handles search queries
- Provides filter options

**ingestion.py**
- Scans data folders
- Extracts metadata from JSON
- Populates database
- Reports progress

**match_service.py**
- Loads match JSON files
- Converts to ball-by-ball CSV
- Manages caching
- Handles exports

**config.py**
- Configuration settings
- Folder mappings
- Path definitions

### Frontend Files

**app.py**
- Main Streamlit application
- Page routing
- Session state management
- UI orchestration

**styles.py**
- Custom CSS styling
- Cricket-themed design
- Modern UI components

**components/filters.py**
- Filter sidebar UI
- Filter logic
- Active filter display

**components/match_table.py**
- Search results display
- Table and card views
- Match selection

**components/match_detail.py**
- Ball-by-ball viewer
- Match info display
- Quick insights

**components/analysis.py**
- cricsummary integration
- Analysis UI
- Chart generation
- Data export
